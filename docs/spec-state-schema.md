# State Schema

Canonical reference for every piece of persistent and semi-persistent state in mclaude. All ADRs, other specs, evaluators, and implementation must be consistent with this document. When a feature adds, removes, or changes state, this document is updated first (in the same commit as the ADR that motivates the change).

The design-evaluator checks all `docs/adr-*.md` and `docs/spec-*.md` files against this schema for consistency. Discrepancies are surfaced as gaps.

---

## Postgres (control-plane)

Single PostgreSQL instance in the control-plane cluster. Managed by `mclaude-control-plane/db.go`.

### `users`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PRIMARY KEY | UUID v4 |
| `slug` | TEXT | UNIQUE NOT NULL | Typed-slug identifier used in subjects, URLs, and KV keys. Derived at user creation by slugifying the full email: `lower(regexp_replace(email, '[^a-zA-Z0-9]+', '-', 'g'))` trimmed of leading/trailing `-`, truncated to 63 chars (ADR-0062). Examples: `dev@mclaude.local` → `dev-mclaude-local`, `richard.song@gmail.com` → `richard-song-gmail-com`. Numeric suffix on collision. Immutable after creation; email changes do not rewrite the slug. |
| `email` | TEXT | UNIQUE NOT NULL | Login email |
| `name` | TEXT | NOT NULL | Display name (free-form UTF-8, max 128 chars, mutable) |
| `password_hash` | TEXT | NOT NULL DEFAULT '' | bcrypt hash (legacy / dev-seed; production users authenticate via OAuth) |
| `oauth_id` | TEXT | NULL | Provider's stable user identifier from the linked OAuth identity. NULL for unlinked rows (e.g. bootstrap-admin row created before first login). On first OAuth callback whose email matches the row, control-plane sets this to the provider's `id`. |
| `nkey_public` | TEXT | UNIQUE | NKey public key. Generated by the user's client (SPA generates in browser `localStorage`, CLI generates at login). Sent to CP during HTTP authentication (`POST /api/auth/challenge`). Used by CP to mint user JWTs. NULL for users who have not yet completed NKey-based auth. |
| `is_admin` | BOOLEAN | NOT NULL DEFAULT FALSE | Gate for `/admin/*` endpoints (cluster register, grant, user management). Set TRUE by the `init-keys` Helm Job for the bootstrap admin email; subsequent admin promotion via `POST /admin/users/{uslug}/promote`. |
| `created_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |

Writers: control-plane (CreateUser, DeleteUser, PromoteAdmin, OAuth-callback link).
Readers: control-plane (GetUserByEmail, GetUserByID, GetUserBySlug, auth middleware, admin gate).

Slug charset: `[a-z0-9][a-z0-9-]{0,62}`, excluding leading `_` and the reserved-word blocklist `{users, hosts, projects, sessions, clusters, api, events, lifecycle, quota, terminal}`. See `docs/adr-0024-typed-slugs.md`. **Known bug:** `computeUserSlug()` in `db.go` does not check against the blocklist — a user with email `api@example.com` gets slug `api`, colliding with the reserved word.

### `projects`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PRIMARY KEY | UUID v4 |
| `user_id` | TEXT | NOT NULL FK→users ON DELETE CASCADE | Owner |
| `slug` | TEXT | NOT NULL | Typed-slug identifier. Unique within the owning user (`UNIQUE (user_id, slug)`). Derived as `slugify(name)` with numeric suffix on collision within scope. Immutable after creation. |
| `name` | TEXT | NOT NULL | Display name (free-form UTF-8, max 128 chars, mutable, e.g. "mclaude") |
| `git_url` | TEXT | NOT NULL DEFAULT '' | Optional git remote |
| `status` | TEXT | NOT NULL DEFAULT 'active' | active, pending, archived |
| `host_id` | TEXT | NOT NULL FK→hosts ON DELETE CASCADE | Host the project is provisioned on (machine or cluster host) |
| `git_identity_id` | TEXT | NULL FK→oauth_connections ON DELETE SET NULL | Optional link to an OAuth connection providing git credentials for this project's repo |
| `source` | TEXT | NOT NULL DEFAULT 'created' | How the project was created: `created` (normal) or `import` (via `mclaude import`). See ADR-0053. |
| `import_ref` | TEXT | NULL | S3 object key for the import archive (e.g. `alice/laptop-a/myapp/imports/imp-001.tar.gz`). Non-NULL only for imported projects while the archive exists in S3. Cleared by CP after the agent signals import completion. See ADR-0053. |
| `created_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |

Index: `UNIQUE (user_id, host_id, slug)` — projects are unique-by-slug per user per host.

Writers: control-plane (CreateProject)
Readers: control-plane (GetProjectsByUser, GetProjectsByHost, GetProjectBySlug), `mclaude-controller-k8s` and `mclaude-controller-local` (via NATS request payload — they receive `host_id` indirectly through the host-scoped subject; they do not query Postgres directly).

### `hosts`

Single source of truth for both BYOH machines and K8s clusters per ADR-0035 (which supersedes ADR-0004 + ADR-0011 + ADR-0014), updated by ADR-0054. The `clusters` table no longer exists. Hosts are globally unique by slug — there is one row per host, not one row per user-host pair. User access to hosts is tracked in the separate `host_access` table.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PRIMARY KEY | UUID v4 |
| `owner_id` | TEXT | NOT NULL FK→users ON DELETE CASCADE | Registering user (permanent owner). Set at registration, immutable. The owner has full control: rename, deregister, revoke credentials, grant/revoke access for other users. |
| `slug` | TEXT | UNIQUE NOT NULL | Typed-slug identifier. Globally unique (one row per host). For machine hosts, user-chosen during `mclaude host register`. For cluster hosts, admin-controlled (the cluster's canonical name). Immutable. |
| `name` | TEXT | NOT NULL | Display name (free-form UTF-8, max 128 chars, mutable) |
| `type` | TEXT | NOT NULL CHECK (`type IN ('machine', 'cluster')`) | `machine` (BYOH laptop) or `cluster` (K8s worker) |
| `nkey_public` | TEXT | UNIQUE | NKey public key. For machine hosts: the host controller's NKey (host-generated during `mclaude host register`; private seed never leaves the machine). For cluster hosts: the cluster controller's NKey (controller-generated at startup). Used by `$SYS` presence to identify which host slug a connection belongs to. All identities generate their own NKeys — CP never handles private key material. |
| `nats_jwt` | TEXT | | Host JWT scoped to `mclaude.hosts.{hslug}.>` with zero JetStream access. NULL until issued by control-plane via HTTP challenge-response. 5-minute TTL; host controller proactively refreshes via the same HTTP auth flow. |
| `created_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |
| `last_seen_at` | TIMESTAMPTZ | | Updated by `$SYS.ACCOUNT.*.CONNECT` subscription on hub NATS. Authoritative historical record. The current `online` boolean lives in `mclaude-hosts` KV, not Postgres. |

Constraints:
- `UNIQUE (slug)` — hosts are globally unique by slug.

Writers: control-plane (`RegisterHost`, `RemoveHost`, `UpdateHostName`, `$SYS` presence subscriber for `last_seen_at`).
Readers: control-plane (`GetHostsByUser`, `GetHostBySlug`, login handler, auth middleware, presence tracking), `mclaude-controller-k8s` (subscribes to `mclaude.hosts.{cluster-slug}.>` and uses cluster slug as a configured value).

Slug charset: per ADR-0024. The reserved-word blocklist is `{users, hosts, projects, sessions, clusters, api, events, lifecycle, quota, terminal, create, delete, input, config, control}` — extended by ADR-0054 with session subject type-tokens. `clusters` stays in the blocklist defensively even though no `mclaude.clusters.*` subjects exist anymore.

#### Default machine host

On user creation, control-plane writes one row to `hosts` with `slug='local'`, `type='machine'`, `owner_id=<new user id>`. The user's first project is associated with this host unless explicitly registered against a different one. **Known bug:** `POST /admin/users` and `seedDev` `CreateUser` do not create the default host row — only the startup migration backfill does. Admin-created users cannot create projects until the binary restarts.

See `docs/adr-0035-unified-host-architecture.md` for the unified host model and `docs/adr-0054-nats-jetstream-permission-tightening.md` for the host access and credential model.

### `host_access`

Per ADR-0054. Tracks which users have access to which hosts. The owner always has implicit access (not stored in this table). Access grants are managed by the host owner via `manage.grant` / `manage.revoke-access` NATS subjects.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `host_id` | TEXT | NOT NULL FK→hosts ON DELETE CASCADE | Host being accessed |
| `user_id` | TEXT | NOT NULL FK→users ON DELETE CASCADE | User granted access |
| `granted_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | When access was granted |

Constraints:
- `PRIMARY KEY (host_id, user_id)` — composite primary key, one grant per user-host pair.

Writers: control-plane (on `manage.grant` / `manage.revoke-access` from host owner).
Readers: control-plane (resolves user's accessible hosts at JWT issuance: owned hosts + granted hosts).

### `agent_credentials`

Per ADR-0054. Tracks registered session-agent NKey public keys. Each agent generates its own NKey pair at startup; the host controller registers the public key with CP via `mclaude.hosts.{hslug}.api.agents.register`. CP stores the public key here and uses it to validate HTTP challenge-response authentication when the agent requests its per-project JWT.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PRIMARY KEY | UUID v4 |
| `user_id` | TEXT | NOT NULL FK→users ON DELETE CASCADE | Owning user |
| `host_slug` | TEXT | NOT NULL | Host where the agent runs |
| `project_slug` | TEXT | NOT NULL | Project the agent manages |
| `nkey_public` | TEXT | UNIQUE NOT NULL | Agent's NKey public key (generated by agent, never by CP) |
| `created_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |

Constraints:
- `UNIQUE (user_id, host_slug, project_slug)` — one credential per project per host per user.

Writers: control-plane (on `mclaude.hosts.{hslug}.api.agents.register` from host controller).
Readers: control-plane (validates agent HTTP challenge-response authentication before minting per-project JWT).

### `oauth_connections`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PRIMARY KEY | UUID v4 |
| `user_id` | TEXT | NOT NULL FK→users ON DELETE CASCADE | Owning user |
| `provider_id` | TEXT | NOT NULL | Admin-configured provider instance ID (from `providers.json`) |
| `provider_type` | TEXT | NOT NULL | `github` or `gitlab` |
| `auth_type` | TEXT | NOT NULL | `oauth` or `pat` |
| `base_url` | TEXT | NOT NULL | Provider API base URL (e.g. `https://github.com`, `https://gitlab.example.com`) |
| `display_name` | TEXT | NOT NULL | Human-readable label (e.g. "GitHub — alice") |
| `provider_user_id` | TEXT | NOT NULL | Provider's stable user identifier |
| `username` | TEXT | NOT NULL DEFAULT '' | Provider username (used to resolve `GIT_IDENTITY_ID` → username in session-agent) |
| `scopes` | TEXT | NOT NULL DEFAULT '' | Granted OAuth scopes |
| `token_expires_at` | TIMESTAMPTZ | NULL | Token expiry (GitLab OAuth tokens expire; GitHub PATs do not) |
| `connected_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |

Constraints:
- `UNIQUE (user_id, base_url, provider_user_id)` — one connection per user per provider identity.

Tokens are stored in per-user K8s Secrets (`user-secrets` in the user namespace), not in Postgres. The `oauth_connections` table stores only metadata. Secret keys follow the pattern `conn-{id}-token`, `conn-{id}-refresh-token`, `conn-{id}-username`.

Writers: control-plane (OAuth callback handler, PAT handler, GitLab token refresh goroutine).
Readers: control-plane (provider listing, credential rotation, connection deletion).

### `attachments`

Per ADR-0053. Tracks binary data (images, screenshots, files) attached to chat messages. S3 stores the actual bytes; this table stores metadata and confirmation state.

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | TEXT | PRIMARY KEY | Opaque attachment ID (e.g. `att-001`), generated by CP on `upload-url` request |
| `s3_key` | TEXT | UNIQUE NOT NULL | Full S3 object key (`{uslug}/{hslug}/{pslug}/attachments/{id}`) |
| `filename` | TEXT | NOT NULL | Original filename (preserved for download) |
| `mime_type` | TEXT | NOT NULL | MIME type (e.g. `image/png`, `application/pdf`) |
| `size_bytes` | BIGINT | NOT NULL CHECK (size_bytes > 0) | File size in bytes |
| `user_id` | TEXT | NOT NULL FK→users ON DELETE CASCADE | Owning user |
| `host_id` | TEXT | NOT NULL FK→hosts ON DELETE CASCADE | Host the project runs on |
| `project_id` | TEXT | NOT NULL FK→projects ON DELETE CASCADE | Project the attachment belongs to |
| `confirmed` | BOOLEAN | NOT NULL DEFAULT FALSE | TRUE after upload confirmation (unconfirmed rows are garbage-collected) |
| `created_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | |

Indexes:
- `idx_attachments_project` on `(project_id)` — lookup for project-level cleanup on project deletion
- `idx_attachments_unconfirmed` on `(confirmed, created_at) WHERE confirmed = FALSE` — garbage collection of abandoned uploads

Writers: control-plane (upload-url handler creates row with `confirmed=false`; confirm handler sets `confirmed=true`)
Readers: control-plane (download handler, project deletion cleanup)

## NATS KV Buckets

All KV bucket keys use typed slugs as their identifier tokens per ADR-0024. Separator is `.` uniformly.

### `mclaude-sessions-{uslug}`

Per-user KV bucket (ADR-0054). Created by control-plane on user registration (`ensureSessionsKV` — `nats.KeyValueConfig{Bucket: "mclaude-sessions-" + uslug}`). The user slug is encoded in the bucket name for NATS-level isolation — each user's JWT can only access their own bucket.

Key format: `hosts.{hslug}.projects.{pslug}.sessions.{sslug}` (hierarchical with literal type-tokens per ADR-0054; user slug is in the bucket name, not the key)

Value: `SessionState`
```json
{
  "id": "string (UUID v4)",
  "slug": "string (session slug)",
  "userSlug": "string",
  "hostSlug": "string",
  "projectSlug": "string",
  "projectId": "string (UUID v4)",
  "branch": "string",
  "worktree": "string",
  "cwd": "string",
  "name": "string",
  "status": "pending | running | paused | requires_action | completed | stopped | cancelled | needs_spec_fix | failed | error",
  "stateSince": "RFC3339",
  "createdAt": "RFC3339",
  "model": "string",
  "backend": "claude_code | droid | devin_acp | gemini | generic_terminal (default: claude_code)",
  "extraFlags": "string (optional — additional CLI flags persisted across restarts, e.g. --disallowedTools, --model)",
  "capabilities": {
    "hasThinking": true,
    "hasSubagents": false,
    "hasSkills": true,
    "hasPlanMode": false,
    "hasMissions": false,
    "hasEventStream": true,
    "hasSessionResume": true,
    "toolIcons": {},
    "thinkingLabel": "Thinking",
    "modelOptions": ["string"],
    "permissionModes": ["string"]
  },
  "tools": ["string"],
  "skills": ["string"],
  "agents": ["string"],
  "pendingControls": { "requestId": { ... } },
  "usage": {
    "inputTokens": 0,
    "outputTokens": 0,
    "cacheReadTokens": 0,
    "cacheWriteTokens": 0,
    "costUsd": 0.0
  },
  "replayFromSeq": 0,
  "joinWorktree": false,
  "softThreshold": 0,
  "hardHeadroomTokens": 0,
  "autoContinue": false,
  "pausedVia": "quota_soft | quota_hard | '' (empty when not paused)",
  "claudeSessionID": "string (Claude Code session ID; for --resume fallback)",
  "branchSlug": "string (matches ^[a-z0-9][a-z0-9-]*$)",
  "failedTool": "string (tool name that triggered needs_spec_fix)",
  "resumeAt": "RFC3339 | null (set on autoContinue-paused sessions; = 5h reset time from QuotaStatus.r5)"
}
```

**Status enum** (renamed from `state` per ADR-0044/ADR-0054 — deployed as clean cut-over):
- `pending` — process alive, awaiting prompt delivery (quota gate)
- `running` — actively processing (includes idle between turns)
- `paused` — quota-paused (soft or hard), subprocess alive
- `requires_action` — blocked on permission or user input
- `completed` — finished successfully
- `stopped` — stopped by user (not cancelled)
- `cancelled` — cancelled, subprocess reaped
- `needs_spec_fix` — permission-denied, spec fix needed
- `failed` — unrecoverable error
- `error` — transient error, may recover

**Backend field** (ADR-0005): Identifies the CLI backend driver. Defaults to `claude_code`. The SPA uses this to toggle per-backend UI features.

**Capabilities struct** (ADR-0005): Contains boolean feature flags from `CLICapabilities` — `hasThinking`, `hasSubagents`, etc. The `tools`, `skills`, `agents` arrays are promoted to **top-level** fields (migrated from the old nested `capabilities.{tools,skills,agents}` struct). `CLICapabilities` is populated from the `init` event.

**Quota fields** (ADR-0044): Optional fields for quota-managed sessions. Interactive sessions omit them (zero values). The SPA can distinguish quota-managed sessions by checking `softThreshold > 0`.

Writers: session-agent (on init, every state change, usage accumulation)
Readers: SPA (KV watch for real-time state), session-agent (recovery on resume)
History: 64 (maximum supported by NATS KV; sufficient for resume tracking)

### `mclaude-projects-{uslug}`

Per-user KV bucket (ADR-0054). Created by control-plane on user registration (`ensureProjectsKV` — `nats.KeyValueConfig{Bucket: "mclaude-projects-" + uslug, History: 1}`). The user slug is encoded in the bucket name for NATS-level isolation.

Key format: `hosts.{hslug}.projects.{pslug}` (hierarchical with literal type-tokens per ADR-0054; user slug is in the bucket name, not the key)

Value: `ProjectState`
```json
{
  "id": "string (UUID v4)",
  "slug": "string",
  "userSlug": "string",
  "hostSlug": "string",
  "name": "string",
  "gitUrl": "string",
  "status": "string",
  "createdAt": "RFC3339",
  "gitIdentityId": "string | null",
  "importRef": "string | null (import ID, e.g. 'imp-001'; set by CP on import, cleared by agent after unpack — see ADR-0053)"
}
```

Note: The spec target includes `sessionCount`, `worktrees`, and `lastActiveAt` fields, but these are not yet implemented in `ProjectKVState` (Go struct). Current Go implementation writes only the 9 fields shown above. The additional fields are planned for a future iteration:
```json
{
  "sessionCount": 0,
  "worktrees": ["string"],
  "lastActiveAt": "RFC3339"
}
```

Writers: control-plane (on project creation), session-agent (project state updates, e.g. clear importRef)
Readers: SPA (KV watch for project list), session-agent
History: 1

### `mclaude-hosts`

Shared KV bucket (ADR-0054). Created by control-plane on deployment (`ensureHostsKV`). Unlike sessions and projects, hosts is a single shared bucket — CP writes once per `$SYS` event. Read access is scoped per-host in the user JWT: each user's JWT lists explicit host slugs they can read (derived from the user's host access list in Postgres at issuance time). On access change, CP revokes the user's JWT and the SPA reconnects with an updated JWT containing the new host list.

Key format: `{hslug}` (flat, no user prefix — hosts are globally unique per ADR-0054)

Value:
```json
{
  "slug": "string",
  "type": "machine | cluster",
  "name": "string",
  "online": true,
  "lastSeenAt": "RFC3339"
}
```

Writers: control-plane only — single writer. Primary path: `$SYS.ACCOUNT.*.CONNECT` writes the full value object (`slug`, `type`, `name`, `online=true`, `lastSeenAt=now`). `$SYS.ACCOUNT.*.DISCONNECT` writes only `{online: false}` — all other fields (`slug`, `type`, `name`, `lastSeenAt`) are preserved from the previous entry (read-modify-write: fetch current value, set `online=false`, re-put). `lastSeenAt` is **not** rewritten on disconnect. Secondary path (dev only): on `DEV_SEED=true` startup, control-plane writes the bootstrap user's `local` machine host entry with `online=true` because the auto-created `local` host has no NKey and will never trigger a `$SYS` CONNECT event.
Readers: SPA (KV watch for host list + status — scoped per-host in user JWT).
History: 1

The previous `mclaude-clusters` KV is removed. SPA derives the per-user cluster list by filtering the `hosts` array in the login response (`hosts.filter(h => h.type === 'cluster')`); no separate KV bucket is needed. **Dead code:** `subj.ClustersKVKey()` in `mclaude-common/pkg/subj/subj.go` and `kvKeyUserClusters()` in `mclaude-web/src/lib/subj.ts` still reference this removed bucket.

The previous `mclaude-laptops` and `mclaude-heartbeats` buckets are removed. Liveness is `$SYS`-only per ADR-0035 (no periodic heartbeat publishes).

**Job queue KV eliminated (ADR-0044):** The `mclaude-job-queue` KV bucket no longer exists. Quota-managed sessions use the session KV (`mclaude-sessions-{uslug}`) with extended fields (`softThreshold`, `hardHeadroomTokens`, `autoContinue`, `pausedVia`, etc.). Callers publish `sessions.create` directly with optional quota fields; the session-agent handles the lifecycle internally. See ADR-0044 for details.

## NATS JetStream Streams

### `MCLAUDE_SESSIONS_{uslug}`

Per-user consolidated sessions stream (ADR-0054). Replaces the previous three shared streams (`MCLAUDE_EVENTS`, `MCLAUDE_API`, `MCLAUDE_LIFECYCLE`). Created by control-plane on user registration. The session-agent no longer creates streams on startup — CP creates them.

```
Name:      MCLAUDE_SESSIONS_{uslug}
Subjects:  mclaude.users.{uslug}.hosts.*.projects.*.sessions.>
Retention: LimitsPolicy
MaxAge:    30 days
Storage:   FileStorage
Discard:   DiscardOld
```

This single stream captures all session activity for a user: events, input/commands, lifecycle, and control messages — all under the consolidated `sessions.>` hierarchy (ADR-0054).

Subject patterns captured:
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.create` — session creation
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.events` — session event stream (canonical `SessionEvent` per ADR-0005)
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.input` — user input
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.delete` — session deletion
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.config` — session config updates
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.control.interrupt` — interrupt
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.control.restart` — restart
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.lifecycle.started` — session started
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.lifecycle.stopped` — session stopped
- `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.lifecycle.error` — session error

Publishers: session-agent (canonical `SessionEvent` events, lifecycle events), SPA (session commands, user input), QuotaMonitor (prompt delivery, soft-stop markers, resume nudges)
Subscribers: SPA (JetStream consumer for live conversation replay — two patterns: (1) dashboard — DeliverNew, lifecycle filter for all sessions; (2) chat view — DeliverAll, per-session filter for full replay + live), session-agent (ordered push consumer for session commands)

## NATS Subjects (Core Pub/Sub)

These are fire-and-forget messages on core NATS (not JetStream). No persistence. All subjects use typed literals between slugs per ADR-0024 (extended by ADR-0035, updated by ADR-0054): every slug is preceded by a reserved word (`users`, `hosts`, `projects`, `sessions`, `clusters`, `api`, `events`, `lifecycle`, `quota`, `terminal`, `create`, `delete`, `input`, `config`, `control`) that names what the following token is. The token `clusters` remains in the slug blocklist defensively but is not used as a subject typer — there are no `mclaude.clusters.*` subjects in the unified host model.

**User-level subjects** (no host scope):

| Subject Pattern | Publisher | Subscriber | Payload |
|----------------|-----------|------------|---------|
| `mclaude.users.{uslug}.quota` | designated per-project agent (`runQuotaPublisher`) | `QuotaMonitor` (per-session, all agents for this user) | `QuotaStatus` JSON — leaf under user scope (not under `.api.`, since quota is a broadcast signal, not a request/reply endpoint). Only the designated agent publishes; CP assigns one agent per user on registration (ADR-0044). |
| `mclaude.users.{uslug}.api.projects.updated` | control-plane | SPA | Broadcast notification that project state has changed (project created, updated, or deleted). SPA invalidates its project list cache on receipt. **Not yet published by code** — SPA helper `subjProjectsUpdated()` exists but no control-plane code publishes to this subject. SPA currently discovers project changes through KV watches instead. |

**Host-scoped subjects** (ADR-0054 — `mclaude.hosts.{hslug}.>` for host/controller subscriptions):

| Subject Pattern | Publisher | Subscriber | Payload |
|----------------|-----------|------------|---------|
| `mclaude.hosts.{hslug}.users.{uslug}.projects.{pslug}.create` | control-plane (fan-out after HTTP validation) | host controller (`mclaude.hosts.{hslug}.>` subscription) | `{userID, userSlug, hostSlug, projectID, projectSlug, gitUrl, gitIdentityId}` |
| `mclaude.hosts.{hslug}.users.{uslug}.projects.{pslug}.delete` | control-plane (fan-out) | host controller | `{userSlug, hostSlug, projectSlug}` |

Project creation flow (ADR-0054): SPA/CLI uses HTTP `POST /api/users/{uslug}/projects`. CP validates (authorization, slug uniqueness, host assignment), creates Postgres record, writes project KV, then CP itself publishes to the fan-out subject. Users do not publish directly to host-scoped subjects.

Host controllers subscribe to `mclaude.hosts.{hslug}.>`:
- `mclaude-controller-k8s` — subscribes to `mclaude.hosts.{cluster-slug}.>`. Receives requests from every user granted access to the cluster.
- `mclaude-controller-local` (BYOH variant, ADR-0058) — subscribes to `mclaude.hosts.{hslug}.>`. Zero JetStream access.

**Hub-side system subjects (control-plane subscribes for liveness):**

| Subject Pattern | Publisher | Subscriber | Payload |
|----------------|-----------|------------|---------|
| `$SYS.ACCOUNT.{accountKey}.CONNECT` | NATS server (hub) | control-plane, host controllers (Sub.Allow per ADR-0054) | Per-connection event; payload includes `client.kind` (`"Client"` / `"Leafnode"`) and `client.nkey` (the connecting client's NKey public key). |
| `$SYS.ACCOUNT.{accountKey}.DISCONNECT` | NATS server (hub) | control-plane, host controllers (Sub.Allow per ADR-0054) | Same shape as CONNECT. |

The NATS server publishes one `$SYS.ACCOUNT.{accountKey}.CONNECT` event per client connection (and one DISCONNECT per drop). Because there is exactly one account in the install, every connection from every component (host controller, SPA, control-plane itself) fires this subject. The control-plane discriminates by `client.kind` and `client.nkey`:

| Event | `client.kind` | Lookup | Effect |
|-------|---------------|--------|--------|
| CONNECT | `Client` | `SELECT * FROM hosts WHERE nkey_public = client.nkey AND type = 'machine'` | If found: update `hosts.last_seen_at = NOW()` for that single row; upsert `mclaude-hosts` KV `{hslug}` with `online=true`. If not found: ignore (covers SPA's per-login NKey, control-plane's own service connection, and any unexpected client). |
| CONNECT | `Leafnode` | `SELECT * FROM hosts WHERE nkey_public = client.nkey AND type = 'cluster' LIMIT 1` | If found: update `hosts.last_seen_at = NOW()` for that row; upsert `mclaude-hosts` KV `{slug}` with `online=true`. |
| DISCONNECT | `Client` | same as above | Set KV `online=false` for the matched row. `last_seen_at` is **not** rewritten on disconnect (it tracks last-known-online). |
| DISCONNECT | `Leafnode` | same as above | Set KV `online=false` for the matching row. |

`$SYS` is the only liveness signal. There is no periodic heartbeat publish; "online" means a NATS connection from that host is currently live. A host controller idle for 5 minutes still shows online until its connection actually drops.

The control-plane's own NATS connection uses an ephemeral user JWT signed by the account key (`claims.Name = "control-plane"`, no explicit publish/subscribe allow-lists — unrestricted within the account). It does not match any row in `hosts.nkey_public`, so the self-CONNECT event is naturally ignored. SPA connections similarly use the per-login NKey (generated by the SPA in browser `localStorage`), which is stored in `users.nkey_public` not `hosts.nkey_public`.

Note: Session subjects (`sessions.create`, `sessions.{sslug}.input`, etc.) are captured by the per-user `MCLAUDE_SESSIONS_{uslug}` stream for at-least-once delivery. The session-agent consumes them via an ordered push consumer (ADR-0054 — replaces the previous pull consumer model).

Note: All agents (BYOH and K8s) connect directly to hub NATS (ADR-0054). Leaf-node topology is removed from scope — agents use the hub NATS URL directly. This simplifies the permission model (one set of subjects, no domain-qualified JetStream).

- StorageClass: from session-agent-template ConfigMap

- Mounted at: `/nix` in session-agent pod

The controller does not read Postgres — control-plane owns Postgres. Session slugs are per-session and flow through NATS messages / KV state; they are not pod env vars.

Writers: `mclaude-controller-k8s` (`reconcileDeployment`)

- ServiceAccount: `mclaude-sa`
- Role: `mclaude-role` — allows get/watch/patch on ConfigMap `user-config` (for config reload) and get on Secret `user-secrets` (for NATS credentials and OAuth tokens mounted into session-agent pods)
- RoleBinding: `mclaude-role` — binds Role to ServiceAccount

Writers: `mclaude-controller-k8s` (`reconcileRBAC`)

## NATS Server Configuration

Per ADR-0035 (updated by ADR-0054) the hub NATS server (on `mclaude-cp` cluster) is configured with a 3-tier operator → account → user trust chain. There is exactly one operator and one account per mclaude install. All agents (BYOH and K8s) connect directly to hub NATS — the previous leaf-node worker topology is removed from scope per ADR-0054.

### Hub NATS (in the `mclaude-cp` Kubernetes cluster)

Static configuration deployed by the `mclaude-cp` Helm chart (`nats-config` ConfigMap):

```
port: 4222                    # client connections
http_port: 8222               # monitoring
websocket.port: 8080          # browser clients
max_payload: 8MB              # large tool results
jetstream.store_dir: /data/jetstream
jetstream.max_file_store: configurable
jetstream.domain: hub         # hub's own JetStream domain

operator: $OPERATOR_JWT       # mounted from K8s Secret mclaude-system/operator-keys
resolver: nats                # full NATS resolver (ADR-0054 — replaces MEMORY for JWT revocation support)
resolver_preload:
  $ACCOUNT_PUBLIC_KEY: $ACCOUNT_JWT

system_account: SYS
```

The hub trusts the operator JWT (self-signed); the operator JWT signs the single account JWT; the account signing key signs every JWT issued by control-plane (user JWTs, host JWTs, session-agent JWTs). The `resolver: nats` configuration (replacing the previous `resolver: MEMORY`) enables runtime JWT revocation — CP can revoke compromised JWTs immediately without restarting the NATS server.

**Leaf-node topology removed from scope (ADR-0054):** All agents (BYOH and K8s) connect directly to hub NATS. The previous worker NATS leaf-node architecture (separate NATS servers per cluster with `leafnodes.remotes` pointing at hub) is removed. This simplifies the permission model — one set of subjects, no domain-qualified JetStream (`$JS.{domain}.API.>`). Leaf-node topology can be added later as opt-in for offline resilience if needed.

### Operator + account NKeys

Operator and account keys live in the K8s Secret `mclaude-system/operator-keys` in the central `mclaude-cp` cluster:

```
Secret: mclaude-system/operator-keys (Opaque, mode 0600)
Keys:
  operatorJwt      — self-signed operator JWT
  operatorSeed     — operator NKey seed
  accountJwt       — account JWT signed by the operator
  accountSeed      — account NKey seed (used by control-plane to sign user JWTs)
  accountPublicKey — account NKey public key (used in NATS resolver_preload config)
  resolverPreload  — formatted resolver_preload entry for NATS config (accountPublicKey: accountJwt)
```

Generated on first deploy by the `mclaude-cp` Helm pre-install Job (`mclaude-cp init-keys`). The Job is idempotent: subsequent deploys check Secret existence and exit without regenerating. The hub NATS pod template references this Secret for `resolver_preload`. The control-plane Deployment also reads the Secret to sign all JWTs (user, host, session-agent).

BYOH machines do **not** receive these keys — they hold only their host JWT. Session-agents receive per-project JWTs issued by control-plane via HTTP challenge-response.

### User JWT permissions (SPA / CLI)

Issued by control-plane via HTTP challenge-response (`POST /api/auth/challenge` + `POST /api/auth/verify`), signed by the account signing key. SPA generates its own NKey pair in browser `localStorage` (shared across tabs), sends public key at login, receives only JWT (no `nkeySeed` in response). ~8h TTL; SPA refreshes every 60s.

Example for user `alice` with access to hosts `laptop-a` and `cluster-a`:

```
Pub.Allow:
  mclaude.users.alice.hosts.*.>                                    # Send commands to own hosts
  _INBOX.>                                                         # Request/reply
  $JS.API.DIRECT.GET.KV_mclaude-sessions-alice.>                   # KV get: own sessions
  $JS.API.DIRECT.GET.KV_mclaude-projects-alice.>                   # KV get: own projects
  $JS.API.DIRECT.GET.KV_mclaude-hosts.$KV.mclaude-hosts.laptop-a   # KV get: per-host (one entry per accessible host)
  $JS.API.DIRECT.GET.KV_mclaude-hosts.$KV.mclaude-hosts.cluster-a
  $JS.API.CONSUMER.CREATE.KV_mclaude-sessions-alice.>              # KV watch: own sessions
  $JS.API.CONSUMER.CREATE.KV_mclaude-projects-alice.>              # KV watch: own projects
  $JS.API.CONSUMER.CREATE.KV_mclaude-hosts.*.$KV.mclaude-hosts.laptop-a    # KV watch: per-host
  $JS.API.CONSUMER.CREATE.KV_mclaude-hosts.*.$KV.mclaude-hosts.cluster-a
  $JS.API.CONSUMER.CREATE.MCLAUDE_SESSIONS_alice.>                 # Session stream consumer
  $JS.API.STREAM.INFO.KV_mclaude-sessions-alice                    # Stream info (KV init)
  $JS.API.STREAM.INFO.KV_mclaude-projects-alice
  $JS.API.STREAM.INFO.MCLAUDE_SESSIONS_alice
  $JS.API.CONSUMER.INFO.KV_mclaude-sessions-alice.*                # Consumer info
  $JS.API.CONSUMER.INFO.KV_mclaude-projects-alice.*
  $JS.API.CONSUMER.INFO.KV_mclaude-hosts.*
  $JS.API.CONSUMER.INFO.MCLAUDE_SESSIONS_alice.*
  $JS.ACK.KV_mclaude-sessions-alice.>                              # Ack consumed messages
  $JS.ACK.KV_mclaude-projects-alice.>
  $JS.ACK.KV_mclaude-hosts.>
  $JS.ACK.MCLAUDE_SESSIONS_alice.>
  $JS.FC.KV_mclaude-sessions-alice.>                               # Flow control
  $JS.FC.KV_mclaude-projects-alice.>
  $JS.FC.KV_mclaude-hosts.>
  $JS.FC.MCLAUDE_SESSIONS_alice.>

Sub.Allow:
  mclaude.users.alice.hosts.*.>                                    # Receive replies from own hosts
  _INBOX.>                                                         # Request/reply
  $KV.mclaude-sessions-alice.hosts.>                               # KV watch: push delivery
  $KV.mclaude-projects-alice.hosts.>
  $KV.mclaude-hosts.laptop-a                                       # KV watch: per-host push delivery
  $KV.mclaude-hosts.cluster-a
  $JS.FC.KV_mclaude-sessions-alice.>                               # Flow control
  $JS.FC.KV_mclaude-projects-alice.>
  $JS.FC.KV_mclaude-hosts.>
  $JS.FC.MCLAUDE_SESSIONS_alice.>
```

Per-user resources (sessions, projects) contain the user slug — no wildcards that match other users' resources. The hosts KV is shared but read access is scoped per-host in the JWT. No `$JS.API.STREAM.INFO.KV_mclaude-hosts` — removed to prevent host enumeration via `SubjectFilter`. No KV write permissions to sessions, projects, or hosts. No `$JS.API.STREAM.DELETE.*`, `$JS.API.STREAM.PURGE.*`, or `$JS.API.STREAM.CREATE.*`. See ADR-0054 for the full threat model.

### Host / controller JWT permissions (BYOH and K8s)

Issued by control-plane via HTTP challenge-response. Host controller generates its own NKey pair. 5-minute TTL; host proactively refreshes. **Zero JetStream access** — hosts use only NATS core pub/sub for provisioning.

```
Pub.Allow:
  mclaude.hosts.{hslug}.>          # Project create/delete (fan-out from CP), agent registration
  _INBOX.>                         # Request/reply

Sub.Allow:
  mclaude.hosts.{hslug}.>          # Subscribe to provisioning and agent registration
  _INBOX.>                         # Request/reply
  $SYS.ACCOUNT.*.CONNECT           # System events (liveness)
  $SYS.ACCOUNT.*.DISCONNECT        # System events (liveness)
```

No `$JS.*`, `$KV.*`, or `$O.*` subjects. This JWT is constant-size regardless of how many users share the host.

### Session-agent JWT permissions (per-project scoped)

Issued by control-plane via HTTP challenge-response (agent generates its own NKey pair; host controller registers the public key via `mclaude.hosts.{hslug}.api.agents.register`). 5-minute TTL; agent proactively refreshes. Scoped to one user, one host, one project.

Example for agent managing alice's project "myapp" on host "laptop-a":

```
Pub.Allow:
  mclaude.users.alice.hosts.laptop-a.projects.myapp.>              # This project's subjects only
  _INBOX.>                                                         # Request/reply
  $KV.mclaude-sessions-alice.hosts.laptop-a.projects.myapp.sessions.>   # KV write: this project's sessions
  $KV.mclaude-projects-alice.hosts.laptop-a.projects.myapp              # KV write: this project's state
  $JS.API.DIRECT.GET.KV_mclaude-sessions-alice.$KV.mclaude-sessions-alice.hosts.laptop-a.projects.myapp.sessions.>   # KV get (subject-form)
  $JS.API.DIRECT.GET.KV_mclaude-projects-alice.$KV.mclaude-projects-alice.hosts.laptop-a.projects.myapp
  $JS.API.DIRECT.GET.KV_mclaude-hosts.$KV.mclaude-hosts.laptop-a   # KV get: this host (read-only)
  $JS.API.CONSUMER.CREATE.KV_mclaude-sessions-alice.*.$KV.mclaude-sessions-alice.hosts.laptop-a.projects.myapp.sessions.>   # KV watch
  $JS.API.CONSUMER.CREATE.KV_mclaude-projects-alice.*.$KV.mclaude-projects-alice.hosts.laptop-a.projects.myapp
  $JS.API.CONSUMER.CREATE.KV_mclaude-hosts.*.$KV.mclaude-hosts.laptop-a
  $JS.API.CONSUMER.CREATE.MCLAUDE_SESSIONS_alice.*.mclaude.users.alice.hosts.laptop-a.projects.myapp.sessions.>   # Session stream (filtered to this project)
  $JS.API.STREAM.INFO.KV_mclaude-sessions-alice                    # Stream info (KV init — per-user, not per-project)
  $JS.API.STREAM.INFO.KV_mclaude-projects-alice

  $JS.API.CONSUMER.INFO.KV_mclaude-sessions-alice.*                # Consumer info

  $JS.API.CONSUMER.INFO.MCLAUDE_SESSIONS_alice.*
  $JS.ACK.KV_mclaude-sessions-alice.>                              # Ack consumed messages
  $JS.ACK.KV_mclaude-projects-alice.>
  $JS.ACK.KV_mclaude-hosts.>
  $JS.ACK.MCLAUDE_SESSIONS_alice.>

  $JS.FC.KV_mclaude-projects-alice.>
  $JS.FC.KV_mclaude-hosts.>
  $JS.FC.MCLAUDE_SESSIONS_alice.>
  mclaude.users.alice.quota                                        # Designated agent publishes quota (ADR-0044)

Sub.Allow:
  mclaude.users.alice.hosts.laptop-a.projects.myapp.>              # This project's subjects only

```

Agent has KV **write** on sessions and projects (it manages state; users are read-only). Agent NATS subjects are scoped to one project (`projects.myapp.>`), not all hosts. No Object Store access — binary data uses S3 (ADR-0053). See ADR-0054 for the full per-project scoping analysis and residual metadata leakage (R4).

## Local File State (BYOH Host)

### `~/.claude/.credentials.json`

```json
{
  "claudeAiOauth": {
    "accessToken": "string"
  }
}
```

Writers: Claude Code (on OAuth login)
Readers: designated per-project agent `readOAuthToken` (for quota API polling)

### Host credentials directory (BYOH machines)

Path: `~/.mclaude/hosts/{hslug}/`

Contents:
- `nkey.seed` — NKey private seed (generated locally on host, never leaves machine; matches `hosts.nkey_public` in Postgres)
- `nats.creds` — NATS credentials file (JWT + NKey seed, host-scoped permissions per ADR-0054 — `mclaude.hosts.{hslug}.>` only, zero JetStream)
- `config.json` — host metadata (`{slug, hubUrl, userSlug}`)

Writers: `mclaude host register` (on registration), host controller JWT refresh loop (HTTP challenge-response, 5-min TTL).
Readers: host controller (`mclaude-controller-local`) NATS connection.

Active-host pointer: `~/.mclaude/active-host` is a symlink to the active `~/.mclaude/hosts/{hslug}/` directory. The host controller and the CLI consult this when `--host` is not provided.

BYOH machines never receive operator or account NKeys; they hold only their host JWT (scoped to `mclaude.hosts.{hslug}.>` with zero JetStream), which is sufficient to authenticate as a NATS client of the hub. Per-project session-agents on BYOH hosts authenticate independently via HTTP challenge-response and receive their own per-project JWTs.

### User-level credentials

Path: `~/.mclaude/auth.json` (mode `0600`)

Contents: bearer token from `mclaude login`, used for admin CLI HTTP calls (`mclaude cluster register`, `mclaude cluster grant`, `mclaude admin users …`). The CLI sends `Authorization: Bearer <token>` on every HTTP call. The token is user-scoped, not host-scoped — it lives outside the per-host directory because the same admin token is valid across all of the user's hosts.

Writers: `mclaude login`.
Readers: CLI (admin and cluster subcommands).

## Login Response Shape

`POST /auth/login` returns a JSON document used by the SPA to bootstrap.

**Current implementation (flat response):**
```json
{
  "natsUrl":   "wss://hub.mclaude.example/nats",
  "jwt":       "<per-user JWT signed by the account key>",
  "userId":    "uuid",
  "userSlug":  "alice-gmail",
  "expiresAt": 1735689600
}
```

The SPA generates its own NKey pair in browser `localStorage` (shared across tabs) and sends the public key during HTTP authentication (`POST /api/auth/challenge`). CP returns only the JWT — no `nkeySeed` in the response (ADR-0054: all identities generate their own NKeys; CP never handles private key material).

The SPA derives host and project data from KV watches (`mclaude-hosts`, `mclaude-projects-{uslug}`) rather than from the login response.

**Spec target (not yet implemented):** The ADR-0035 target includes a richer response with nested `user` object, `hosts[]` array (full host inventory), and `projects[]` array. This enables the SPA to bootstrap without waiting for KV watches:

```json
{
  "user":     { "id": "uuid", "slug": "alice-gmail" },
  "jwt":      "<per-user JWT signed by the account key>",
  "hubUrl":   "wss://hub.mclaude.example/nats",
  "hosts": [
    {
      "slug":        "mbp16",
      "name":        "alice's MBP",
      "type":        "machine",
      "role":        "owner",
      "online":      true,
      "lastSeenAt":  "RFC3339"
    },
    {
      "slug":          "us-east",
      "name":          "Production K8s",
      "type":          "cluster",
      "role":          "user",
      "online":        true,
      "lastSeenAt":    "RFC3339",
      "jsDomain":      "us-east",
      "directNatsUrl": "wss://us-east.mclaude.example/nats"
    }
  ],
  "projects": [
    { "slug": "myrepo",  "name": "My Repo",        "hostSlug": "mbp16",   "hostType": "machine" },
    { "slug": "billing", "name": "billing service","hostSlug": "us-east", "hostType": "cluster", "jsDomain": "us-east", "directNatsUrl": "wss://us-east.mclaude.example/nats" }
  ]
}
```

When implemented, the `hosts` array will be the single source of truth for host inventory. SPA will derive the cluster list with `hosts.filter(h => h.type === 'cluster')`. `jsDomain` and `directNatsUrl` will be present only on cluster-type entries — the SPA includes JetStream domain qualification only when these fields are present, so single-host BYOH deployments work unchanged.

## Lifecycle Event Payloads

Published to `mclaude.users.{uslug}.hosts.{hslug}.projects.{pslug}.sessions.{sslug}.lifecycle.{eventType}` (ADR-0054 target — lifecycle events live under the session namespace, captured by `MCLAUDE_SESSIONS_{uslug}` stream). These are the event types and their payloads:

### `session_created`
```json
{ "type": "session_created", "sessionId": "...", "branch": "...", "ts": "RFC3339" }
```
**Known bug:** code uses the generic `publishLifecycle()` which only emits `{type, sessionId, ts}` — the `branch` field is not included.

### `session_stopped`
```json
{ "type": "session_stopped", "sessionId": "...", "exitCode": 0, "ts": "RFC3339" }
```
**Known bug:** code uses the generic `publishLifecycle()` which only emits `{type, sessionId, ts}` — the `exitCode` field is not included.

### `session_restarting`
```json
{ "type": "session_restarting", "sessionId": "...", "ts": "RFC3339" }
```

### `session_resumed`
```json
{ "type": "session_resumed", "sessionId": "...", "ts": "RFC3339" }
```

### `session_failed`
```json
{ "type": "session_failed", "sessionId": "...", "error": "...", "ts": "RFC3339" }
```

### `debug_attached`
```json
{ "type": "debug_attached", "sessionId": "...", "ts": "RFC3339" }
```

### `debug_detached`
```json
{ "type": "debug_detached", "sessionId": "...", "ts": "RFC3339" }
```

### `session_upgrading`
```json
{ "type": "session_upgrading", "sessionId": "...", "ts": "RFC3339" }
```
Published during graceful shutdown (zero-downtime upgrade) after all sessions reach idle.

### `session_permission_denied`
```json
{ "type": "session_permission_denied", "sessionId": "...", "tool": "...", "ts": "RFC3339" }
```

### `session_job_complete`
```json
{ "type": "session_job_complete", "sessionId": "...", "branch": "...", "ts": "RFC3339" }
```

Published by: QuotaMonitor's `handleTurnEnd()` when the session ended naturally (`stopReason == ""`). No `jobId` (ADR-0044 eliminates the job concept — sessions are the single abstraction). No `prUrl` (ADR-0044 eliminates stdout-marker scanning).

### `session_job_paused`
```json
{
  "type": "session_job_paused",
  "sessionId": "...",
  "pausedVia": "quota_soft | quota_hard",
  "u5": 76,
  "r5": "RFC3339",
  "outputTokensSinceSoftMark": 12345,
  "ts": "RFC3339"
}
```

Published by: QuotaMonitor's `handleTurnEnd()` on soft-stop turn-end OR hard-stop interrupt. `pausedVia` distinguishes them. `r5` is used for `autoContinue` resume scheduling.

### `session_job_cancelled`
```json
{ "type": "session_job_cancelled", "sessionId": "...", "ts": "RFC3339" }
```

Published by: session-agent `handleDelete` for quota-managed sessions (`softThreshold > 0`). Agent interrupts subprocess, publishes this event, then tombstones the session KV entry.

### `session_job_failed`
```json
{ "type": "session_job_failed", "sessionId": "...", "error": "...", "ts": "RFC3339" }
```

Published by: QuotaMonitor's `handleSubprocessExit()` when the subprocess exited without a prior terminal event (no `handleTurnEnd()` completion or pause).

## Quota Status

Published to `mclaude.users.{uslug}.quota` (core NATS, not JetStream).

```json
{
  "hasData": true,
  "u5": 42,
  "r5": "RFC3339",
  "u7": 15,
  "r7": "RFC3339",
  "ts": "RFC3339"
}
```

Writers: designated per-project agent `runQuotaPublisher` (every 60s, polls `api.anthropic.com/api/oauth/usage`). CP designates one agent per user on registration (ADR-0044); if the designated agent goes offline, CP re-designates another.
Readers: `QuotaMonitor` per-session goroutine (NATS subscription) in all per-project agents for this user
