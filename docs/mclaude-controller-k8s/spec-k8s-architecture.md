# Spec: mclaude-controller-k8s

The K8s controller is a kubebuilder operator that runs inside a worker Kubernetes cluster. It owns per-project runtime resources (Namespace, Deployment, PVCs, Secrets) for all users whose sessions are hosted on that cluster.

The controller connects directly to hub NATS via JWT credential (host-scoped per ADR-0054). There is no local NATS StatefulSet; the worker chart no longer ships a leaf node.

## Deployment

Single Deployment in `mclaude-system` on the worker cluster. Built with kubebuilder (controller-runtime). Leader election flag configurable via `LEADER_ELECTION` env var. v1 runs a single replica.

## Boot Sequence

1. Read the NKey seed from the file at `HOST_NKEY_SEED_PATH` (default `/etc/mclaude/host-creds/nkey_seed`). The seed was generated by the `gen-host-nkey` pre-install Job and stored in the `{release}-host-creds` Secret.
2. Call `NewHostAuthFromSeed(seedPath, cpURL, log)` from `mclaude.io/common/pkg/hostauth`. This returns a `HostAuth` handle with no JWT yet.
3. Call `Refresh()` to perform HTTP challenge-response (`POST /api/auth/challenge` + `POST /api/auth/verify`) against `CONTROL_PLANE_URL`. If CP returns 404 (NKey not registered — operator has not yet run `mclaude host register`), log:
   ```
   NKey <pubkey> not registered with control-plane. To complete registration run:
     mclaude host register --type cluster --name <display-name> --nkey-public <pubkey>
   ```
   and retry every 5s with exponential backoff capped at 60s. Controller does not proceed until JWT is acquired.
4. Decode the host slug (`hslug`) from the received JWT. Use this slug for all subsequent NATS subscriptions — do not read `host.name` from Helm.
5. Connect to hub NATS at `HUB_NATS_URL` using the host JWT.
6. Subscribe to `mclaude.hosts.{hslug}.users.*.projects.*.create` and `mclaude.hosts.{hslug}.users.*.projects.*.delete`.
7. Background loop refreshes the host JWT before TTL expiry (5-minute TTL) via the same challenge-response code path.

## Registration Flow (One-Time Operator Action)

The `gen-host-nkey` pre-install Job (weight `-10`) generates an NKey pair at `helm install` time, writes the decorated seed string to Secret `{release}-host-creds` data field `nkey_seed`, and prints the public key to the Job log and Helm NOTES.txt.

The operator must attest the host before the controller can boot:

```bash
# Read pubkey from Job log
kubectl logs job/{release}-gen-host-nkey -n mclaude-system

# Attest from workstation (operator's user JWT as attestation)
mclaude host register --type cluster --name "us-east" --nkey-public <pubkey>
```

This publishes `mclaude.users.{uslug}.hosts._.register {name, type, nkey_public}` over NATS. CP slugifies `--name`, creates a `hosts` row keyed by `public_key`, returns `{ok, slug}`. No new API surface — same flow BYOH machines use.

## Deregistration Flow

Two options; both publish `mclaude.users.{uslug}.hosts.{hslug}.manage.deregister` per ADR-0054, which causes CP to soft-delete the `hosts` row and add the host JWT to the NATS revocation list:

**Option A (operator-initiated):**
```bash
mclaude host deregister --slug us-east   # publishes deregister from workstation
helm uninstall mclaude-worker -n mclaude-system
```

**Option B (Helm pre-delete Job):**
The `mclaude-worker` Helm chart includes a pre-delete hook Job that uses the host's stored NKey seed to perform challenge-response auth and then publishes the deregister subject before Helm tears down the controller. This ensures the host is deregistered even if the operator forgets to run `mclaude host deregister` first.

In both cases, all MCProject CRs and their owned resources (Deployments, PVCs, Secrets) are cleaned up by `helm uninstall` via `ownerReferences` cascade or Helm-tracked resource deletion.

## Configuration

| Variable | Required | Description |
|----------|----------|-------------|
| `HUB_NATS_URL` | Yes | Hub NATS WebSocket URL (e.g. `nats-wss://nats.mclaude.example:443`). |
| `CONTROL_PLANE_URL` | Yes | Control-plane HTTP base URL for challenge-response auth and session-agent bootstrap. |
| `HOST_NKEY_SEED_PATH` | No | Path to the NKey seed file (default `/etc/mclaude/host-creds/nkey_seed`). |
| `HELM_RELEASE_NAME` | No | Used to locate the session-agent-template ConfigMap (default `mclaude-worker`). |
| `SESSION_AGENT_TEMPLATE_CM` | No | Explicit name of the session-agent-template ConfigMap. Overrides `HELM_RELEASE_NAME`-derived name. |
| `DEV_OAUTH_TOKEN` | No | Claude API OAuth token for dev environments. Injected as `oauth-token` in per-user `user-secrets` Secret. |
| `METRICS_ADDR` | No | Prometheus metrics listen address (default `:8082`). |
| `HEALTH_PROBE_ADDR` | No | Health/readiness probe listen address (default `:8081`). |
| `LEADER_ELECTION` | No | Enable leader election (default `false`). |

Dropped from prior spec (no longer used):
- `NATS_ACCOUNT_SEED` — controller no longer holds the account signing key
- `NATS_CREDENTIALS_PATH` — no leaf-creds file
- `JS_DOMAIN` — no JetStream domain qualification
- `CLUSTER_SLUG` — slug is decoded from the JWT after challenge-response
- `SESSION_AGENT_NATS_URL` / `NATS_URL` — session-agents inherit `HUB_NATS_URL` directly

## NATS Subscriptions

Subscribes via host JWT (`mclaude.hosts.{hslug}.>`, zero JetStream access):

| Subject | Behavior |
|---------|----------|
| `mclaude.hosts.{hslug}.users.*.projects.*.create` | Request/reply. CP fan-out provisioning (ADR-0054). Resolves `userSlug`, `hostSlug`, `projectSlug` from subject + payload; creates the `MCProject` CR; returns success when reconcile reaches `Ready`. |
| `mclaude.hosts.{hslug}.users.*.projects.*.delete` | Request/reply. Deletes the `MCProject` CR; owner references cascade to project Deployment/PVC/Secret. |

## Kubernetes Resources

For full schemas see `docs/spec-state-schema.md` — Kubernetes Resources. Summary:

- CRD `MCProject` (`mcprojects.mclaude.io/v1alpha1`) in `mclaude-system`.
- Per-user namespace `mclaude-{userSlug}` (ADR-0062) with ServiceAccount, Role, RoleBinding.
- Per-user `user-secrets` Secret (OAuth tokens, CLI configs — **not** NATS credentials; see Session-Agent Auth).
- Per-user `user-config` ConfigMap.
- Per-project `project-{projectId}` and `nix-{projectId}` PVCs.
- Per-project `project-{projectId}` Deployment with `ownerReferences` back to the MCProject CR.
- Watched: `{release}-session-agent-template` ConfigMap in `mclaude-system` — changes re-enqueue all `MCProject` CRs.

All Deployments, PVCs, and Secrets materialized per project carry `ownerReferences` back to the `MCProject` CR via `controllerutil.SetControllerReference`. Deleting the `MCProject` CR cascades to all owned resources.

## Reconciler Loop

On each reconcile cycle:

1. Loads the session-agent-template ConfigMap for image, resource, and PVC configuration.
2. Ensures the user namespace `mclaude-{userSlug}` exists with correct labels (ADR-0062: namespace derived from `userSlug`, not UUID).
3. Ensures RBAC resources (ServiceAccount, Role, RoleBinding).
4. Ensures the `user-config` ConfigMap and `user-secrets` Secret. The `user-secrets` Secret contains `oauth-token` (from `DEV_OAUTH_TOKEN` if set) and OAuth connection tokens. It does **not** contain `nats-creds` — session-agent pods self-bootstrap NATS credentials via challenge-response (see Session-Agent Auth below).
5. Copies imagePullSecrets from the controller's namespace.
6. Ensures the project PVC and Nix PVC (with `ownerReferences` back to MCProject).
7. Ensures the session-agent Deployment (with `ownerReferences` back to MCProject) using `Recreate` strategy. Pod env vars include `USER_ID`, `USER_SLUG`, `HOST_SLUG`, `PROJECT_ID`, `PROJECT_SLUG`, `NATS_URL` (= `HUB_NATS_URL`), `CONTROL_PLANE_URL` (injected from the controller's own `CONTROL_PLANE_URL` env, not from the session-agent-template ConfigMap).
8. Updates `MCProject` status: phase (`Pending → Provisioning → Ready` or `Failed`) and conditions.

## Session-Agent Auth

The controller no longer mints session-agent JWTs. Session-agent pods self-bootstrap per ADR-0054:

1. Agent generates its own NKey pair at pod startup.
2. Agent passes its public key to the controller via local IPC.
3. Controller publishes `mclaude.hosts.{hslug}.api.agents.register {nkey_public, userSlug, hostSlug, projectSlug}` to hub NATS; CP stores the key in `agent_credentials`.
4. Agent calls `POST /api/auth/challenge` + `POST /api/auth/verify` to obtain its per-project JWT.
5. Agent connects to hub NATS (`HUB_NATS_URL`) with its JWT.

The `nats-creds` Secret key is not written by `reconcileSecrets`. The `user-secrets` Secret retains only `oauth-token` and OAuth connection entries.

## Corporate CA Support

When the session-agent-template ConfigMap has `corporateCAEnabled: "true"`, the controller:
- Adds label `mclaude.io/user-namespace: "true"` to user namespaces for trust-manager targeting.
- Injects a `corporate-ca` volume, volume mount at `/etc/ssl/certs/corporate-ca-certificates.crt`, and `NODE_EXTRA_CA_CERTS` env var.
- Annotates the pod template with `mclaude.io/ca-bundle-hash` so CA rotations trigger a `Recreate` rollout.

## Liveness

When the controller connects to hub NATS, hub publishes `$SYS.ACCOUNT.{accountKey}.CONNECT` with `client.kind = "Client"`. Control-plane's `sys_subscriber.go` looks up the host by `public_key` (no type filter — matches both `type='machine'` and `type='cluster'`), updates `hosts.last_seen_at`, and upserts `mclaude-hosts` KV `online=true`. On disconnect, `online=false`.

## Error Handling

| Failure | Behavior |
|---------|----------|
| `HOST_NKEY_SEED_PATH` file missing at boot | Fatal exit — Secret not mounted correctly. Kubernetes restarts the pod. |
| CP returns 404 on challenge-response (key not registered) | Log registration instruction with pubkey; retry every 5s (exponential backoff, cap 60s). Pod remains in boot loop until operator runs `mclaude host register`. K8s liveness probe deadline will crashloop the pod if operator never acts. |
| Hub NATS unreachable at boot (after JWT acquired) | Retry with NATS client backoff. No provision requests processed during gap. |
| Hub NATS drops at runtime | NATS client reconnects; existing JWT used until TTL; on TTL expiry, challenge-response refresh. If CP unreachable at refresh time, controller exits and K8s restarts. |
| MCProject reconcile fails | MCProject CR enters `Failed` phase; CP surfaces error to user via NATS. |
| Namespace creation fails (RBAC, quota) | MCProject CR enters `Failed` phase; logs detail. |
| ownerReference cascade fails | Should not occur with `controllerutil.SetControllerReference` in place. If observed, file a bug — owner refs are the cleanup path. |
| Operator uses different `--name` at registration than `host.name` Helm value | CP keys off public key, not name. Controller decodes actual slug from JWT. `host.name` value is informational (NOTES.txt only). |

## Helm Chart Artifacts

Deployed by `charts/mclaude-worker`. See `docs/charts-mclaude/spec-helm.md` for full Helm resource table.

| Secret | Field | Contents |
|--------|-------|---------|
| `{release}-host-creds` | `nkey_seed` | Decorated NKey seed string (`SUAB...`). Written by `gen-host-nkey` Job at install time. JWT is **not** stored here — acquired in-memory at boot. |

## Dependencies

- Hub NATS reachable from the worker cluster (via `HUB_NATS_URL` Ingress at 443).
- Control-plane HTTP reachable (for challenge-response).
- Kubernetes API for Namespace, Deployment, PVC, Secret, ConfigMap, RBAC, and `MCProject` CRD.
- The `{release}-session-agent-template` ConfigMap deployed by `mclaude-worker` Helm chart.
- `mclaude.io/common/pkg/hostauth` for shared NKey challenge-response logic.
- Optional: trust-manager for corporate CA bundle injection.
