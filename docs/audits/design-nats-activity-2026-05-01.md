1. **`import.complete` payload contradicts ADR-0053** — Section 11d shows the agent publishing an empty payload `{}` for `import.complete`. ADR-0053 (the authoritative source for imports, referenced as a dependency via ADR-0058) specifies the standard message envelope: `{"id":"...","ts":...}` (no additional fields). The payload schema (`spec-nats-payload-schema.md`) shows a third variant with `s3Key`, `bytesDownloaded`, `filesExtracted`. Three specs disagree. A developer implementing the agent's `import.complete` publisher would not know which payload to use.
   - **Doc**: Section 11d — `Agent → NATS: PUB ...import.complete  {}`

The payload schema `import.complete` section shows `{"id":"01JTRK8MH...","ts":1714470087000,"s3Key":"...","bytesDownloaded":1048576,"filesExtracted":42}`.

2. **Quota publisher `Pub.Allow` not reflected in ADR-0054** — Section 7d explicitly states the designated quota publisher agent's JWT must include `mclaude.users.alice.quota` in `Pub.Allow`. ADR-0054 (listed as a dependency for the permission model) does not include this entry in the session-agent Pub.Allow spec. ADR-0054's "Deferred" section says: "Quota system redesign — The current quota subject (`mclaude.users.{uslug}.quota`) has no `.hosts.` or `.projects.` tokens and is incompatible with the new permission model. Deferred to a future ADR." The activity spec resolves this deferral (sections 7d, 8e, 8f, 9i describe the full mechanism), but ADR-0054 hasn't been updated to reflect the resolution. A developer implementing JWT issuance from ADR-0054's permission spec alone would produce JWTs that lack the quota publish permission, causing NATS permission violations when the designated agent tries to publish quota updates. ADR-0044 also describes the quota publisher mechanism but similarly does not specify the JWT Pub.Allow entry.
   - **Doc**: Section 7d — "Agent JWT Pub.Allow also includes (for designated quota publisher only): `mclaude.users.alice.quota` (publish quota updates — see section 9g)"
   - **Code**: ADR-0054 session-agent Pub.Allow lists only `mclaude.users.alice.hosts.laptop-a.projects.myapp.>` — the per-project scope. `mclaude.users.alice.quota` is at the user level, outside this wildcard. ADR-0054 Deferred section explicitly says quota permission is deferred.
