The session-agent currently creates all three streams on startup (`agent.go:101-140`) and creates pull consumers on `MCLAUDE_API` for at-least-once session command delivery (create, input, delete, control, restart). Similarly, `MCLAUDE_LIFECYCLE` is used for session lifecycle event persistence and consumption by the daemon and SPA. Both streams have the same cross-user access concerns as `MCLAUDE_EVENTS` but are not migrated to per-user streams or given scoped permissions.
